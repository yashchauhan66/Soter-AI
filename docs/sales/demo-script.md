# Demo Script

1. State the limitation: no tool detects every AI attack or model failure.
2. Show a safe request passing through the input guard.
3. Show a synthetic prompt-injection example being rewritten or blocked.
4. Show PII or secret redaction using fictional data.
5. Show output inspection and the safe fallback.
6. Open logs, policy, signed webhook delivery, and the monthly report.
7. For RAG, show quarantine, approval, ACL filtering, and grounding fallback.
8. Close with integration effort, plan limits, and the next pilot step.

Never paste production credentials, customer logs, or unredacted personal data into a demo.
