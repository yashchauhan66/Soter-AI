# Objection Handling

## The Model Provider Already Has Safety

Provider controls are useful. CyberRakshak Guard adds application-specific policy, redaction, provider-independent evidence, alerts, and input/output controls around the full workflow.

## Can You Guarantee Security?

No. CyberRakshak Guard is OWASP LLM Top 10 aligned and supports defense-in-depth risk reduction. Secure design, identity, authorization, human review, monitoring, and incident response remain necessary.

## We Have Not Had an Incident

The decision should not rely on fear. Evaluate the data exposed, buyer requirements, current visibility, and cost of building equivalent controls. A free authorized audit can establish whether a real gap exists.

## False Positives Will Hurt Users

Start in monitor or balanced mode, use approved evaluation cases, review feedback, and move to stricter policy only when the team accepts the tradeoff.

## Integration Will Add Latency

Measure p50/p95 in the customer's path and agree on a budget. The pilot should fail if the operational target is not met.

## It Is Too Expensive

Compare the price with custom engineering, manual review, delayed enterprise deals, and support effort. If no measurable value exists, do not force the sale.

## We Need a New Feature First

Ask whether the feature blocks the agreed security outcome and whether multiple qualified customers need it. Avoid custom roadmap commitments based on one conversation.

## We Need Certification

The product offers alignment mapping and evidence, not customer certification. Clarify which exact procurement/control requirement must be met.
